# Agents

Agents let p2pstream forward public traffic into networks that do not accept inbound connections.

## Identity

Agent IDs and tokens are generated by the management UI. The token is shown once when the agent is created or rotated.

Store the token as a secret. If it leaks, rotate the token from the Agent Health page and update the agent service.

## Connection model

Agents connect outbound to the management server:

- HTTPS for stats and API calls,
- WSS for request forwarding.

By default, agents reject insecure HTTP management URLs and verify HTTPS certificates.

## Install methods

The Agent Setup dialog can generate:

- a one-line Linux systemd installer,
- a Docker Compose service,
- a direct CLI command.

The installer downloads the release binary, verifies the checksum, writes `/etc/p2pstream/agent.env`, and enables `p2pstream-agent.service`.

## TLS trust

For auto-generated management TLS, pass one of:

| Variable | Use |
| --- | --- |
| `MANAGEMENT_CA_FILE` | Path to a PEM CA bundle on the agent host. |
| `MANAGEMENT_CA_PEM_BASE64` | Base64-encoded PEM CA bundle, useful for generated snippets. |

If management requires agent client certificates, configure both:

```text
AGENT_TLS_CERT_FILE=/etc/p2pstream/agent.crt.pem
AGENT_TLS_KEY_FILE=/etc/p2pstream/agent.key.pem
```

The client certificate must identify the agent with the expected p2pstream URI.

## Agent health

The Agent Health page shows:

- connected/offline state,
- active requests,
- latest memory and goroutine stats,
- last connected time,
- token rotation and enable/disable actions.
